# #31 가족 정보 화면 — 멤버 목록 및 초대 코드 통합

## 개요

현재 더보기 > "가족 정보" 는 탭 시 초대 코드를 클립보드에 복사하고 Alert만 띄우는 단순 액션이다.
누가 가족에 연동되어 있는지 볼 방법이 없고, 관련 기능들이 흩어져 있다.

**연계 이슈:**
- #29 가족 탈퇴 → 가족 정보 화면 내 탈퇴 버튼으로 진입
- #9 초대 코드 개선 → 만료일 표시·재생성 버튼을 이 화면에서 제공
- #3 멤버별 지출 분리 → 멤버 목록이 있어야 UI에서 구분 가능

---

## 화면 구성

### FamilyInfoScreen (신규)

더보기 > 가족 정보 탭 시 → `FamilyInfoScreen`으로 navigate

```
┌─────────────────────────────┐
│  ← 가족 정보                │  (네비게이션 헤더)
├─────────────────────────────┤
│  멤버 (2명)                 │  (섹션 헤더)
│  ┌──────────────────────┐   │
│  │ 🖼 상운              │   │  (프로필 사진 + 이름)
│  │ 🖼 채원              │   │
│  └──────────────────────┘   │
├─────────────────────────────┤
│  초대 코드                   │  (섹션 헤더)
│  ┌──────────────────────┐   │
│  │  3F8A2C1B   [복사]   │   │  (#9 적용 전: 기존 inviteCode 표시)
│  │  3월 15일까지 유효    │   │  (#9 적용 후: 만료일 표시)
│  │  [코드 재생성]        │   │  (#9 적용 후)
│  └──────────────────────┘   │
├─────────────────────────────┤
│  [가족 탈퇴]                 │  (danger 색상, #29 연계)
└─────────────────────────────┘
```

### 멤버 목록

- `family.members` 배열 순회
- 각 uid의 프로필: `users/{uid}` 에서 `displayName`, `photoURL` 조회
- 프로필 사진 없으면 이니셜 아바타 (MoreMenuScreen의 기존 아바타 컴포넌트 재사용)
- 본인 항목에 "(나)" 표시

### 초대 코드 섹션

**#9 적용 전 (현재):**
- 기존 `family.inviteCode` 문자열 표시
- 복사 버튼

**#9 적용 후:**
- `inviteCodes` 컬렉션에서 활성 코드 조회
- 만료일 표시
- "코드 재생성" 버튼

→ #9 미완료 시점에도 이 화면을 먼저 만들 수 있도록, 초대 코드 섹션은 기존 방식으로 구현 후 #9 완료 시 교체.

## Firestore 데이터 모델

변경 없음.

멤버 목록 표시 시 `users/{uid}` 를 멤버 수만큼 조회 (현재 최대 2명).
React Query로 캐싱하여 중복 요청 방지.
