### 인터랙션 정의

| 트리거 | 동작 |
|--------|------|
| Header [+] 탭 | RecurringFormScreen 신규 모드 push |
| RecurringItem 탭 | RecurringFormScreen 수정 모드 push, `params.recurring` 전달 |
| [반영] 탭 | `useApplyRecurring()` 호출 → 성공: 미반영 목록 제거 + 토스트 "반영됐어요" / 실패: 에러 토스트 |
| 토글 변경 | `useUpdateRecurring({ isActive })` 즉시 호출 |
| 스와이프 삭제 | Alert("정기 지출을 삭제하시겠어요?", "삭제된 고정비는 복구되지 않아요") → 확인 시 `useDeleteRecurring()` |

## 4. 화면 전환 플로우

```
HomeScreen
  │  (배너 [확인하기] 탭)
  ▼
RecurringListScreen ←──── MoreMenuScreen ("정기 지출 관리" 탭)
  │
  ├─ [+] 버튼 탭 ──────────▶ RecurringFormScreen (신규)
  │                               │ 저장 → pop
  │
  └─ RecurringItem 탭 ────────▶ RecurringFormScreen (수정)
                                  │ 수정 → pop
```
